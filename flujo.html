Flujo básico

  1. Crear o abrir el proyecto.
  2. Inicializar Git con git init.
  3. Agregar archivos con git add ..
  4. Guardar cambios con git commit.
  5. Subir al repositorio remoto con git push.

Ejemplo real

git init
git add .
git commit -m "Agrega estructura inicial del sitio"
git branch -M main
git remote add origin https://github.com/usuario/repositorio.git
git push -u origin main

Push

Envía los commits locales al repositorio remoto.

Pull

Trae los cambios del remoto al equipo local.

Clone

Copia un repositorio remoto completo a tu computadora.

Trabajar con ramas

git checkout -b feature/navbar
git add .
git commit -m "Mejora la navegación"
git push origin feature/navbar

El flujo correcto evita pérdidas de trabajo y facilita la colaboración.
